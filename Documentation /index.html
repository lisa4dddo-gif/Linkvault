What is LinkVault?

LinkVault is a self-hosted, premium link monetization tool. It allows you to place any target URL (a download link, a referral link, a file, or any webpage) behind a countdown timer with integrated ads. Visitors must wait out the timer — during which your ads run — before they can access the destination. This turns your existing traffic into a predictable ad revenue stream.

The project is architected to be entirely configuration-driven. You never need to touch the core source code. All customization is done through the visual Admin Tool (included), which generates the config files for you.

⚡ Quick Setup — The Admin Tool

This is the most important section. The Admin Tool is a visual interface included in the /admin/ folder. It lets you configure everything about your site and auto-generate the config files — no coding or text editor required.

💡
No Coding Required

The Admin Tool is a self-contained HTML file. Simply open it in any modern browser (Chrome, Firefox, Edge) directly from your computer. You do not need a server to run it.

1

Open the Admin Tool

Navigate to the admin/ folder inside your downloaded project. Double-click index.html to open it in your browser. You will see a beautiful dashboard with a multi-step configuration form.

2

Fill in Your Site Settings

Complete the Site Configuration form with your details:

  • Site Name — e.g. LinkVault or your own brand name
  • Site URL — your live domain, e.g. https://yourdomain.com
  • Google Analytics ID — e.g. G-XXXXXXXXXX (get this from analytics.google.com)
  • Countdown Timer Duration — in seconds (default: 15–30 seconds recommended)
  • Default Destination URL — the URL visitors are sent to after the timer
3

Paste Your Ad Codes

In the Ad Configuration section of the Admin Tool, paste your ad network scripts (from Adsterra, PropellerAds, etc.) into the appropriate slots — Banner Ad, Popunder, Native Widget, etc. See the Ad Network Setup section for where to get these codes.

4

Click "Generate Files"

Once all fields are filled, click the Generate Config Files button. The Admin Tool will produce two downloadable files: site-config.js and ads.js.

5

Replace the Default Files

Take the two downloaded files and replace the existing placeholder files in your project:

  • Replace assets/js/site-config.js with your new site-config.js
  • Replace assets/js/ads.js with your new ads.js

That's it — your site is now fully configured and ready to upload.

6

Upload to Your Web Host

Upload all project files (except the admin/ folder) to your web hosting via FTP, cPanel File Manager, or your preferred method. The project works on any standard web host — shared hosting, VPS, Netlify, Vercel, GitHub Pages, etc.

⚠️
Delete the /admin/ Folder Before Going Live

The admin folder exposes your configuration interface to the public internet. See the Security Notes section.

Locking a Link (How It Works)

To monetize a specific link, you direct your audience to locked.html with the target URL as a query parameter:

URL Format 
https://yourdomain.com/locked.html?url=https://example.com/your-target-link

Visitors who click this link will see the countdown timer and ads. Once the timer expires, they are automatically redirected to https://example.com/your-target-link. You earn ad revenue from every visit.

Step-by-Step: Getting Your Ad Code

1

Create an Account & Add Your Site

Register on your chosen ad network and add your domain (e.g. yourdomain.com) as a publisher property. Most networks will ask you to verify ownership by uploading a file or adding a meta tag — follow their specific instructions.

2

Create an Ad Zone / Unit

Inside the ad network dashboard, create a new Ad Zone (Adsterra terminology) or Ad Unit. Choose the ad type you want (e.g. "Banner 728x90" or "Social Bar" or "Popunder"). Set the size and targeting options they offer.

3

Copy the JavaScript Snippet

Once the ad unit is created, the network will provide you a <script> tag. It will look something like this:

Example Ad Snippet (Adsterra) 
<script type="text/javascript">
  atOptions = {
    'key' : 'your_unique_ad_key_here',
    'format' : 'iframe',
    'height' : 90,
    'width' : 728,
    'params' : {}
  };
</script>
<script type="text/javascript"
  src="//www.highperformanceformat.com/your_key/invoke.js">
</script>

Copy this entire snippet from the ad network.

4

Paste into the Admin Tool

Open the admin/index.html file, find the Ad Configuration section, and paste your snippet into the correct slot (Banner, Popunder, etc.). The Admin Tool will wrap it properly inside ads.js when you generate the files.

🌐 Translations & Language

The entire user-facing text of LinkVault is stored in a single file: assets/js/lang.js. Translating the app into any language is as simple as editing this one file.

How the Translation System Works

The lang.js file exports a JavaScript object where each key corresponds to a UI element (e.g. page_title_index, btn_get_link). script.js reads this object on page load and populates all text elements. You never edit the HTML files.

assets/js/lang.js — Example Structure 
// ══ LinkVault Language File ══
// To translate: change the VALUES (right side of the colon).
// Do NOT change the KEYS (left side) — they are used by the app.

window.LV_LANG = {

  // ── Page Titles ──
  page_title_index  : "LinkVault — Monetize Every Click",
  page_title_locked : "Get Your Link — LinkVault",

  // ── Homepage ──
  hero_headline     : "Monetize Every Click",
  hero_sub          : "Lock your links. Earn from every visit.",
  btn_get_link      : "Get Link",
  placeholder_url   : "Paste your destination URL here...",

  // ── Countdown Page ──
  countdown_heading : "Your link is being prepared",
  countdown_note    : "Please wait while ads load. Thank you for your support!",
  btn_skip          : "Skip Ad",
  btn_open_link     : "Open My Link →",

  // ── Adblock Warning ──
  adblock_title     : "Ad Blocker Detected",
  adblock_message   : "Please disable your ad blocker to continue.",

  // ... (many more keys inside the file)

};

How to Translate (Step-by-Step)

1

Duplicate the File

Make a copy of assets/js/lang.js and rename it to include your language code, for example: lang.es.js for Spanish, lang.fr.js for French, lang.ar.js for Arabic.

2

Edit the Values

Open your new file in any text editor (Notepad, VS Code, etc.). Translate only the values (the text inside the quotes on the right side of the colon). Never change the keys (the words on the left side).

Example — Spanish Translation 
// BEFORE (English):
hero_headline : "Monetize Every Click",

// AFTER (Spanish):
hero_headline : "Monetiza Cada Clic",
3

Update the Script Tag in index.html and locked.html

Open both index.html and locked.html. Find this line near the top and change the filename to your translated file:

HTML — Change Language File 
<!-- BEFORE -->
<script src="assets/js/lang.js"></script>

<!-- AFTER (Spanish) -->
<script src="assets/js/lang.es.js"></script>

The entire UI will now display in your chosen language. No other changes needed.

📌
Right-to-Left (RTL) Languages

For Arabic, Hebrew, Urdu, or other RTL languages, add dir="rtl" to the <html> tag in index.html and locked.html. The stylesheet already includes RTL-safe layouts.

📱 PWA & Sitemap

Progressive Web App (PWA)

LinkVault is fully PWA-ready. This means visitors can add your site to their phone's home screen and use it like a native app — with a custom icon, splash screen, and offline support.

Step 1 — Edit manifest.json

Open web-app/manifest.json and update the following fields to match your brand:

web-app/manifest.json 
{
  "name":             "Your Site Name",
  "short_name":       "ShortName",
  "description":      "Lock your links and earn from every visit.",
  "start_url":        "/",
  "scope":            "/",
  "display":          "standalone",
  "theme_color":      "#080c14",
  "background_color": "#080c14",
  "icons": [
    {
      "src":   "../assets/img/icon-192.webp",
      "sizes": "192x192",
      "type":  "image/webp"
    },
    {
      "src":   "../assets/img/icon-512.webp",
      "sizes": "512x512",
      "type":  "image/webp"
    }
  ]
}
FieldWhat to Put
nameYour full site name shown on splash screen
short_nameShort name for the home screen icon label (max ~12 chars)
descriptionA one-line description of your site
theme_colorBrowser toolbar color — keep as #080c14 or change to your brand color
iconsReplace the icon files with your own logo at 192×192 and 512×512 pixels

Step 2 — Service Worker (HTTPS Required)

The Service Worker file (web-app/sw.js) enables offline caching and is what makes the site installable. It only activates on HTTPS. Make sure your web host provides an SSL certificate (all major hosts include free SSL via Let's Encrypt).

⚠️
HTTPS is Required for PWA Features

The Service Worker (sw.js) will not register on plain http:// connections. Ensure your hosting has SSL enabled. This is free on virtually all modern hosts (Hostinger, Namecheap, Netlify, Vercel, etc.).

Sitemap Generator

The included generate-sitemap.js script automatically scans your project for all HTML files and creates a complete sitemap.xml — ready to submit to Google Search Console.

📋
Prerequisite: Node.js

You need Node.js installed on your computer to run this script. Download it free from nodejs.org. Any version 14+ works. This is a one-time local task — Node.js does NOT need to be on your server.

Step 1 — Configure the Script

Open generate-sitemap.js in a text editor and update the CONFIG block at the top of the file:

generate-sitemap.js — CONFIG Block 
var CONFIG = {
  /* ★ Change this to your live domain */
  SITE_URL:           'https://yourdomain.com',

  /* Root directory of your project (leave as '.' if running from root) */
  ROOT_DIR:           '.',

  /* Output filename */
  OUTPUT_FILE:        'sitemap.xml',

  /* Folders to skip (admin is excluded for security) */
  EXCLUDE_PATHS:      ['admin', 'node_modules'],

  /* Default values for all pages */
  DEFAULT_CHANGEFREQ: 'weekly',
  DEFAULT_PRIORITY:   '0.7',
  HOMEPAGE_PRIORITY:  '1.0',
};

Step 2 — Run the Script

Open your terminal (Command Prompt, PowerShell, or Terminal on Mac/Linux), navigate to your project folder, and run:

Terminal 
# Navigate to your project folder
cd path/to/linkvault

# Run the sitemap generator
node generate-sitemap.js

A sitemap.xml file will be created in your project root. You will see a success message confirming how many pages were found.

Step 3 — Submit to Google Search Console

  1. Upload sitemap.xml to your live website root (e.g. https://yourdomain.com/sitemap.xml).
  2. Go to search.google.com/search-console and add your property.
  3. Click Sitemaps in the left menu, enter sitemap.xml, and click Submit.
  4. Google will begin crawling and indexing your pages.

🛡️ Security Notes

🚨
CRITICAL: Delete the /admin/ Folder Before Going Live

The /admin/ folder contains a tool that generates your configuration files. If left publicly accessible on your live server, anyone who finds it can view and overwrite your site settings. This is a serious security risk.

Option A — Delete the Folder (Recommended)

The simplest and safest option. After you have generated your site-config.js and ads.js files, simply delete the entire admin/ folder before uploading your project to the server. Keep a local backup on your computer.

Option B — Password Protect with .htaccess

If you want to keep the admin panel accessible on your server for future edits, protect it with a server-level password. This requires Apache hosting (cPanel, Hostinger, etc.):

admin/.htaccess — Basic Auth Protection 
AuthType Basic
AuthName "Admin Area — Restricted"
AuthUserFile /home/yourusername/public_html/admin/.htpasswd
Require valid-user

Create an .htpasswd file using an online .htpasswd generator and place it in the same admin/ folder.

Option C — Block via Nginx Config

On Nginx servers (VPS, DigitalOcean, etc.), add this to your server block:

nginx.conf 
location /admin/ {
    deny all;
    return 403;
}

General Security Checklist

  • Delete or protect the /admin/ folder before going live.
  • Use HTTPS (SSL) — required for PWA, boosts SEO, encrypts visitor data.
  • Keep your Google Analytics ID private — do not share it publicly.
  • Regularly update your ad network code when your provider sends new snippets.
  • Do not commit ads.js or site-config.js to public GitHub repositories.

🔑 Admin Mode (Owner Testing)

LinkVault includes a built-in Admin Mode that allows you (the site owner) to browse your live site safely — without triggering your own analytics or accidentally clicking your own ads. This protects your Google Analytics data and your ad network account from self-click violations.

Activating Admin Mode

Visit your live site with the ?admin=true query parameter appended to the URL:

Activate Admin Mode 
https://yourdomain.com/?admin=true

You will see a browser alert confirming "🛡️ Admin Mode Activated!". From that point on, as long as you use the same browser:

  • All ad zones are hidden (no accidental self-clicks).
  • Google Analytics tracking is disabled for your sessions.
  • A console message confirms the protection is active.
  • Your preference is saved in localStorage — it persists across page refreshes.

Deactivating Admin Mode

Deactivate Admin Mode 
https://yourdomain.com/?admin=false

You will see a confirmation alert. Your session will return to normal visitor mode.

💡
Localhost is Automatically Protected

When running the site on localhost, 127.0.0.1, or a local network IP (192.168.x.x), Admin Mode is always active automatically. You never need to add ?admin=true during local development.

🔍 SEO & Social Meta Tags

LinkVault comes pre-built with a comprehensive SEO setup inside index.html and locked.html. You simply need to update a few placeholders with your own details.

What's Already Configured

  • Standard meta tags (description, author, robots).
  • Canonical URL tag to prevent duplicate content issues.
  • Open Graph tags (Facebook, WhatsApp, LinkedIn sharing previews).
  • Twitter / X Card tags (large image preview cards).
  • PWA theme color and Apple touch icon.

What You Need to Update

Search for the placeholder text https://yourdomain.com in both HTML files and replace it with your actual domain. Do the same for @yourtwitterhandle. Comments in the code guide you through each one.

OG Cover Image

Create a 1200×630 pixel image featuring your brand/logo on a dark background and save it as assets/img/og-cover.webp. This image appears as the preview thumbnail when your link is shared on social media.

index.html — Key Placeholders to Update 
<!-- 1. Canonical URL -->
<link rel="canonical" href="https://yourdomain.com/index.html" />

<!-- 2. Open Graph -->
<meta property="og:url" content="https://yourdomain.com/index.html" />
<meta property="og:image" content="https://yourdomain.com/assets/img/og-cover.webp" />

<!-- 3. Twitter / X -->
<meta name="twitter:site" content="@yourtwitterhandle" />
<meta name="twitter:creator" content="@yourtwitterhandle" />
